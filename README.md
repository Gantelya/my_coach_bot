## 📌 О проекте

**Iron Corner** — это полноценный AI-ассистент для боксёров, построенный на базе LLaMA 3.3 70B через Groq API. Бот ведёт персонализированный диалог с каждым пользователем, помнит всю историю общения, составляет индивидуальные планы тренировок и следит за питанием.

> 💡 Проект создан с нуля: от идеи до деплоя в продакшн на облачной платформе Railway с подключённой базой данных Redis.

-----

## ✨ Функциональность

### 🤖 AI-тренер

- Персонализированные ответы на основе профиля пользователя
- Память диалога — бот помнит всю историю переписки (Redis)
- Составление планов тренировок: мешок, лапы, бой с тенью, ОФП
- Генерация PDF-плана тренировок и питания на неделю

### 🍽 Дневник питания

- Пользователь описывает еду словами
- AI автоматически рассчитывает КБЖУ (калории, белки, жиры, углеводы)
- Оценивает подходит ли питание для спортсмена
- Сохраняет все приёмы пищи за день

### 📊 Прогресс трекер

- Еженедельная запись веса и результатов
- AI анализирует динамику и сравнивает с предыдущей неделей
- Хранение истории за 12 недель
- Мотивационный анализ прогресса от AI-тренера

### ⏰ Умные напоминания

- Пользователь задаёт удобное время для тренировки
- Ежедневные push-уведомления в Telegram
- Фоновая задача работает независимо (asyncio)

### 👨‍💼 Панель администратора

- Статистика пользователей через `/stats`
- Массовая рассылка через `/broadcast`
- Логирование всех сообщений в приватный Telegram-канал

## 🏗 Архитектура проекта

```
iron_corner_bot/
│
├── main.py              # Основной файл бота
│   ├── HealthCheck      # HTTP-сервер для мониторинга
│   ├── Redis Layer      # Функции работы с базой данных
│   ├── AI Layer         # Запросы к Groq API с retry-логикой
│   ├── FSM States       # Состояния для многошаговых диалогов
│   ├── Handlers         # Обработчики сообщений и команд
│   └── reminder_loop()  # Фоновая задача напоминаний
│
└── requirements.txt     # Зависимости проекта
```

### Ключевые архитектурные решения:

- **Асинхронность** — весь код построен на `async/await`, что обеспечивает высокую производительность
- **FSM** — конечный автомат для управления многошаговыми диалогами (ввод времени, прогресса, питания)
- **Retry-логика** — автоматические повторные попытки при ошибках AI с экспоненциальной задержкой
- **Ограничение контекста** — хранение последних 20 сообщений для оптимизации токенов

## 🚀 Установка и запуск

### Требования

- Python 3.11+
- Redis сервер
- Telegram Bot Token ([@BotFather](https://t.me/BotFather))
- Groq API Key ([console.groq.com](https://console.groq.com))

### Локальный запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/username/my_coach_bot.git
cd my_coach_bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Задать переменные окружения
export TELEGRAM_TOKEN="your_token"
export GROQ_API_KEY="your_groq_key"
export REDIS_URL="redis://localhost:6379"

# 4. Запустить бота
python main.py
```

### Деплой на Railway

```bash
# 1. Создать проект на railway.app
# 2. Подключить GitHub репозиторий
# 3. Добавить Redis через Database → Add Redis
# 4. Задать переменные окружения в Variables:
#    TELEGRAM_TOKEN, GROQ_API_KEY
#    REDIS_URL добавится автоматически
```

### Переменные окружения

|Переменная      |Описание                |Обязательная|
|----------------|------------------------|:----------:|
|`TELEGRAM_TOKEN`|Токен бота от @BotFather|✅           |
|`GROQ_API_KEY`  |API ключ от Groq        |✅           |
|`REDIS_URL`     |URL подключения к Redis |✅           |
|`LOG_CHANNEL_ID`|ID канала для логов     |❌           |
